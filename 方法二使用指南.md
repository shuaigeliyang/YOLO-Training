# 方法二完整指南：使用自定义YAML配置集成CBAM和ASFF

## 📋 概述

方法二通过注册自定义模块到ultralytics,然后在YAML配置文件中使用这些模块,实现CBAM和ASFF的集成。这种方法不需要修改ultralytics源码。

## ✅ 优势

1. **不修改源码** - 保持ultralytics原始代码不变
2. **可配置性强** - 通过YAML轻松调整网络结构
3. **易于维护** - 模块独立,方便更新
4. **灵活性高** - 可以自由组合不同模块

## 📁 已创建的文件

### 核心文件

1. **register_custom_modules.py** ⭐
   - 注册CBAM和ASFF模块到ultralytics
   - 创建YAML配置文件
   - 提供测试功能

2. **yolo11n_cbam.yaml**
   - YOLO11n + CBAM配置
   - 在backbone的关键层添加CBAM
   - 适合提升特征表达

3. **yolo11n_cbam_asff.yaml**
   - YOLO11n + CBAM + ASFF完整配置
   - Backbone使用CBAM
   - Neck使用ASFF融合

4. **train_yaml_cbam.py**
   - 完整的训练脚本
   - 自动注册模块并开始训练
   - 包含日志记录

```python
    data='./data.yaml',

    batch=16,
    device=0

```

## 📊 配置文件说明

### yolo11n_cbam.yaml 结构

```yaml
# Backbone - 添加CBAM增强特征
backbone:
  - [-1, 1, Conv, [64, 3, 2]]
  - [-1, 1, Conv, [128, 3, 2]]
  - [-1, 2, C3k2, [256, False, 0.25]]
  - [-1, 1, CBAM, [256]]  # ← CBAM增强
  - [-1, 1, Conv, [256, 3, 2]]
  - [-1, 2, C3k2, [512, False, 0.25]]
  - [-1, 1, CBAM, [512]]  # ← CBAM增强
  ...
```

**CBAM位置选择原则:**
- ✓ 在C3k2模块后添加
- ✓ 在关键特征提取层后
- ✓ 避免过多使用(增加计算量)

### 自定义配置示例

如果要调整CBAM位置:

```yaml
# 更激进的CBAM配置 - 每个C3k2后都加
backbone:
  - [-1, 2, C3k2, [256, False, 0.25]]
  - [-1, 1, CBAM, [256]]  # CBAM
  - [-1, 1, Conv, [256, 3, 2]]
  - [-1, 2, C3k2, [512, False, 0.25]]
  - [-1, 1, CBAM, [512]]  # CBAM
  ...

# 保守的CBAM配置 - 只在关键层
backbone:
  ...
  - [-1, 2, C3k2, [512, True]]
  - [-1, 1, CBAM, [512]]  # 只在深层加
  ...
```

## 🔧 参数调整

### 修改类别数

编辑YAML文件中的`nc`参数:
```yaml
nc: 559  # 改为你的类别数
```

### 调整模型大小

修改`scales`参数:
```yaml
scales:
  n: [0.50, 0.25, 1024]  # nano
  s: [0.50, 0.50, 1024]  # small
  m: [0.75, 0.75, 768]   # medium
```

### CBAM参数

在YAML中可以指定CBAM参数:
```yaml
- [-1, 1, CBAM, [256, 16, 7]]  # [channels, reduction_ratio, kernel_size]
```

## 🐛 故障排查

### 问题1: 模块未找到

**错误:**
```
AttributeError: module 'ultralytics.nn.modules' has no attribute 'CBAM'
```

**解决:**
1. 确保先运行 `register_custom_modules()`
2. 检查cbam_module.py在当前目录
3. 重新运行注册脚本

### 问题2: YAML解析失败

**错误:**
```
KeyError: 'CBAM'
```

**解决:**
1. 确保在加载YAML前注册了模块
2. 检查YAML格式是否正确
3. 使用绝对路径加载模块

### 问题3: 训练过程中显存不足

**解决:**
1. 减小batch size
2. 减少CBAM使用数量
3. 使用混合精度训练(`amp=True`)

## 💡 最佳实践

### 1. 渐进式添加CBAM

**第一步:** 只在backbone深层添加
```yaml
- [-1, 2, C3k2, [512, True]]
- [-1, 1, CBAM, [512]]  # 仅此处
```

**第二步:** 扩展到中层
```yaml
- [-1, 2, C3k2, [256, False, 0.25]]
- [-1, 1, CBAM, [256]]  # 新增
...
- [-1, 2, C3k2, [512, True]]
- [-1, 1, CBAM, [512]]  # 保留
```

**第三步:** 全面使用
```yaml
# 每个特征提取层后都加
```

### 2. 平衡性能和速度

**注重精度:**
```yaml
# 多个CBAM + 大reduction_ratio
- [-1, 1, CBAM, [256, 8]]  # reduction_ratio=8
```

**注重速度:**
```yaml
# 少量CBAM + 小reduction_ratio  
- [-1, 1, CBAM, [256, 16]]  # reduction_ratio=16
```

### 3. 组合使用建议

| 场景 | 推荐配置 |
|------|---------|
| 小目标检测 | Backbone全CBAM |
| 大目标检测 | 仅深层CBAM |
| 多尺度检测 | CBAM + ASFF |
| 实时检测 | 最少CBAM |

## 📝 训练配置建议

```python
# 针对CBAM优化的训练配置
config = {
    'data': './data.yaml',
    'epochs': 300,
    'imgsz': 640,
    'batch': 16,  # 根据显存调整
    
    # 优化器
    'optimizer': 'AdamW',  # AdamW更适合CBAM
    'lr0': 0.01,
    'lrf': 0.01,
    'weight_decay': 0.0005,
    
    # 学习率调度
    'cos_lr': True,
    'warmup_epochs': 3,
    
    # 数据增强 - 与CBAM配合
    'hsv_h': 0.015,
    'hsv_s': 0.7,
    'hsv_v': 0.4,
    'mosaic': 1.0,
    'mixup': 0.1,
    
    # 其他
    'patience': 50,
    'amp': True,  # 混合精度训练
    'verbose': True,
}
```
